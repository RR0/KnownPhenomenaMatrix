Redéfinition Et non "surcharge" .

Motivation

Redéfinir l'implémentation d'une méthode dans une sous-classe.

Exemples

Un exemple de redéfinition est :

class InputStream
{
public close ()
{
...
}
}

class FilterInputStream extends InputStream
{

public close () // Redéfinition de la méthode close()
{
...
}
}

Notes

  • Une redéfinition peut consister à ajouter un comportement au comportement de l'ancêtre (appel de la méthode super() de cet ancêtre).
  • A ne pas confondre avec surcharge (overloading).
